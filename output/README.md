# output/

`bbr-review` スキルが生成したレビュー結果を保存するディレクトリ。

## ファイル名規則

```
review_YYYYMMDD_HHMMSS.md
```

例: `review_20260506_083045.md` （2026-05-06 08:30:45 に実行されたレビュー）

スキルは実行のたびに新規ファイルを書き出すため、**過去のレビュー履歴がそのまま残ります**。「直す → 再レビュー」ループで前回との差分を確認したい場合に活用できます。

## 出力されるレビュー結果の構造

レビュー結果は以下の構造で出力されます（詳細は `.claude/skills/bbr-review/SKILL.md` の「出力フォーマット」参照）。

- TL;DR（総合スコア / レディネス / 報告蓋然性 / 処理結果確率分布 / 品質汚染 / 規約違反リスク / 最重要指摘 3 つ）
- 推定情報（プログラム / 脆弱性タイプ / CVSS / CWE / 攻撃シナリオ要約）
- 観点別スコア（13 観点のテーブル）
- 攻撃シナリオの再構成（Attacker → Victim → System）
- 詳細レビュー（13 観点それぞれの良い点 / 問題点 / 直し方の具体例）
- 提出前チェックリスト
- 提出可否の最終判定

## メンテナンス

不要になった古いレビュー結果は適宜削除してかまいません。スキル本体は `output/` の中身に依存していません。
