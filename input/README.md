# input/

レビュー対象のレポート下書きと、付随する画像（スクリーンショット等）をこのディレクトリに配置します。

## 配置するファイル

- **レポート本文**: `report.md` / `draft.md` / `*.md` / `*.txt`（Markdown 推奨）
  - Markdown ファイルが複数ある場合、`report.md` または `draft.md` を優先。なければ最終更新日時が新しいものを主レポートとして扱います。
  - 残りの Markdown / テキストは補助資料（PoC スクリプト、補足メモ等）として参照されます。
- **画像ファイル**: `*.png` / `*.jpg` / `*.jpeg` / `*.gif` / `*.webp`
  - Burp のリクエスト/レスポンスのスクリーンショット、アラート画面、レスポンスの view-source、PoC 動画のサムネイル等
  - すべて Supporting Material として読み込まれ、観点 11（Supporting Material）の評価に反映されます。

## 注意

- **PII（個人識別情報）を含むスクリーンショットは絶対に置かないでください。**
  - 他人の本名、住所、電話番号、決済情報、メール本文等が写っている画像は、レビュー前に**完全マスキング** or **テスト用データへの差し替え**が必要です。
  - Intigriti 等のプラットフォームでは「実 PII の検証」自体が規約違反対象です。レビューでも観点 11 / 13 でゼロ点になります。
- このディレクトリの中身は**レビュー時にすべて読み込まれます**。古いレポートの残骸が紛れていると誤判定の元になるため、レビューしたいレポート以外は削除しておいてください。

## レビュー実行

ファイルを配置した状態で、Claude Code に以下のいずれかを伝えるとスキルが起動します。

```
input のレポートをレビューして
```

```
/bbr-review
```

レビュー結果は `output/review_YYYYMMDD_HHMMSS.md` として保存されます。
